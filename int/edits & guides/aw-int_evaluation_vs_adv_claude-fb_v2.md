# Comparative Evaluation: *Administrative Writing, Intermediate*
## Against the Advanced Book Development Methodology
### Updated after review of `aw-int_unit_structural_profiles.md`

---

> **Purpose of this document**
> This report evaluates the intermediate book (`aw-int-all.md`) against the four priority dimensions agreed for this pass: (1) pedagogical structure, (2) language instruction quality, (3) prompt and task design, and (4) activity variety and progression. Results are presented as a checklist-style pass/fail assessment with findings and required actions.
>
> **Important note on scope:** This evaluation covers the live book content (`aw-int-all.md`) and the structural profiles document (`aw-int_unit_structural_profiles.md`) together. Several findings from the initial evaluation have been revised upward in light of the profiles document, which resolves two of the original hard failures. Remaining issues are now concentrated in the live book content rather than the planning architecture.

---

## SECTION 1 — Pedagogical Structure (Can-Do Architecture, Activity Sequence)

---

### 1.1 Module introductions are present and coherent

**Result: PASS**

All six modules have a Module Guide section in the live book. Each guide states module-level learning outcomes and gives usage instructions in appropriate intermediate-level language. The profiles document adds a corresponding Module Purpose statement for each module that correctly characterises the module's scope and relationship to adjacent modules.

---

### 1.2 Unit-level Can-Do statements are present

**Result: PASS (revised from FAIL)**

The initial evaluation flagged this as a hard failure based on the live book alone, where the `### Focus` section uses instructional bullet lists rather than learner-facing Can-Do statements. However, the structural profiles document includes 3–4 *"I can…"* Can-Do statements for every unit across all 23 units.

The current situation is therefore: **the Can-Do architecture exists in the profiles but has not yet been transferred into the live book content.** This is a valid staged approach — consistent with how the advanced book was developed (profiles first, content later).

**Remaining action required:** The Can-Do statements from the profiles must be written into the live module files, replacing or supplementing the current `### Focus` bullet lists. This is a content-transfer task, not a redesign task.

---

### 1.3 Activity-level Can-Do statements are present

**Result: PASS (revised from FAIL)**

Again, the initial evaluation flagged this as a hard failure based on the live book alone. The structural profiles document includes a detailed `Activity-purpose map` for every unit, recording the purpose of each activity in the unit sequence.

The activity-purpose maps are not framed as learner-facing *"I can…"* statements — they use third-person descriptions (e.g., "`Guided Practice`: writes one short structured explanation"). They are planning-voice rather than learner-voice. This is slightly weaker than the advanced book's activity-level Can-Do format, but functionally equivalent as a planning tool.

**Remaining action required:** The activity-purpose maps can remain in their current planning-voice format in the profiles document. If a learner-facing unit outline is added to the live book at a later stage (as noted in the profiles as a future step), the activity-purpose entries should be converted to *"I can…"* format at that point.

---

### 1.4 The activity sequence visibly develops the unit goal

**Result: PASS with reservation**

The profiles confirm and explain the unit-level activity sequence for all 23 units. The sequence is pedagogically coherent and the profiles explicitly note the approved exceptions (e.g., Unit 3's front-loaded concept sections, Unit 8's hybrid procedure + manual-section structure).

The reservation from the initial evaluation stands: the sequence is highly consistent across all 23 units (Warm-Up → Model Check → Language → Guided Practice → Freer Practice → Editing → Transfer → Homework). This is appropriate at intermediate level, but units in Modules 4–6 that target more complex skills would benefit from modest variation in activity type — a point noted in the profiles' content flags for several units.

---

### 1.5 Module-level progression is coherent

**Result: PASS**

The profiles confirm the module progression: sentence control (M1) → email writing (M2) → internal documents (M3) → external/audience-facing communication (M4) → editing and revision (M5) → integrated capstone (M6). Each Module Purpose statement in the profiles correctly characterises the relationship to adjacent modules.

---

### 1.6 Unit-level skill progression within each module

**Result: PASS with reservation**

The profiles confirm logical within-module unit sequencing across all six modules. Content flags in several units note where later revision should check whether the step-up in difficulty between adjacent units is sufficient (e.g., Module 2 Units 5 and 7 are flagged for their similarity in request-writing focus).

---

### 1.7 Section H ordering is consistent

**Result: PASS**

The profiles confirm that every unit ends with a Reflection and Homework section. The output-length expectations for homework are clearly recorded for every unit. No ordering inconsistencies are present.

---

### 1.8 The profiles document is structurally complete and fit for use as a planning source of truth

**Result: PASS with one active concern**

All 23 units have complete profile entries covering all 13 required content items listed in the document header. The INT-5A support-depth overlay is present and correctly frames the remaining intervention requirements.

**Active concern — profiles not yet synchronised with live book:** The profiles record Can-Do statements, activity-purpose maps, output expectations, and content flags that are not yet visible in the live book content. The live `### Focus` sections still use the old instructional bullet-list format. Until the profiles are reflected in the live book, there is a gap between planning intent and learner experience. This is the single most important structural task remaining.

## SECTION 2 — Language Instruction Quality (N10-Style Teaching Gaps)

---

### 2.1 The principle behind each target feature is named and explained

**Result: PASS**

Every unit includes a `### Model Check: Why the Stronger [X] Works` section and `**Why this works**` blocks in Language sections. This meets the N10 standard for explicit principle-teaching. Consistent and genuinely effective across all 23 units.

---

### 2.2 Conceptual distinctions between related forms are explained

**Result: PASS with reservation**

Unit 7's politeness scale is the book's strongest example of explicit contrast teaching. The profiles note that this table-format teaching is intentional and should be preserved. Other strong examples exist in Module 5 (the accuracy → clarity → tone editing order).

Reservations that remain from the initial evaluation:
- The register difference between *because* and *due to* is not addressed despite both appearing in practice items.
- In Unit 3, the distinction between a *concluding* sentence and a *forward-linking* sentence is blurred in the model paragraph. The profiles flag this unit as needing later revision attention but do not specifically name this issue.

---

### 2.3 Worked examples support the explanation

**Result: PASS**

`Mini contrast` and `Mini model` blocks are consistently used throughout the book. The profiles explicitly require model/example support in the INT-5A overlay questions, confirming this is an intended structural feature. The profiles also record `model/example support expectation` as a required profile field, confirming that this standard applies to every unit.

---

### 2.4 Transfer is addressed

**Result: PASS**

The `Transfer Extension` appears in every unit and is confirmed as a required structural element in the profiles. The profiles' INT-5A overlay explicitly asks "Where will the unit provide transfer / extension support?" as one of six required intervention checks. Transfer is therefore not only present but treated as a non-negotiable planning requirement.

---

### 2.5 Phrase banks and useful-language sections are explained rather than listed

**Result: PASS with reservation**

The profiles flag the Module 2 phrase-bank section (Unit 2) as an approved exception that needs later decision on whether to absorb it into the standard pattern. Module 4 phrase banks (Units 12–13) are not flagged in the profiles, but the live book content shows that these present phrases without a worked mini-example showing how context + question + closing operate together. This remains a live-book quality issue not captured in the current profile flags.

**Recommended addition to profiles:** Add a content flag to Units 12 and 13 noting that the phrase banks need at least one worked mini-inquiry before the next revision pass.

---

### 2.6 The bridge from explanation to production is visible

**Result: PASS**

`Self-Check Before Freer Practice` appears in every unit and is confirmed as a required structural element. The INT-5A overlay explicitly requires review/self-edit support for every unit. This is well-implemented and consistent throughout the live book.

---

## SECTION 3 — Prompt and Task Design (PAST-R, Calibration, Parallelism)

---

### 3.1 Writing tasks are situation-anchored, not topic-anchored

**Result: PARTIAL PASS**

The profiles acknowledge this issue in content flags for several units. For example, Unit 5's flag notes that "the freer layer is now limited to 2 short emails so the total production load stays more controlled" — but it does not explicitly address the topic-label problem in the Freer Practice options ("Deadline confusion," "Missing attachment," etc.).

The live book still contains topic-anchored Freer Practice options in Units 5, 6, 7, and several Module 5 units. These are not flagged for correction in the current profiles.

**Required action:** Add content flags to Units 5, 6, 7 (Module 2) and Units 16, 17, 18 (Module 5) explicitly requiring that topic-label Freer Practice options be replaced with situation-anchored prompts before the next revision pass.

---

### 3.2 PAST-R elements are recoverable from the prompt

**Result: PARTIAL PASS**

Guided Practice tasks across all modules are well-specified. Homework tasks are generally PAST-R compliant from Module 3 onwards. The Freer Practice tasks with topic-label options (see 3.1) remain non-compliant because Subject and Role are absent.

The profiles record output expectations for every unit but do not systematically audit PAST-R compliance for each task. This is a gap in the profiles' coverage of prompt quality.

---

### 3.3 The prompt does not pre-solve the writing task

**Result: PASS**

Well-calibrated throughout. Guided Practice tasks provide structure and frame without pre-solving content decisions. Mini models are brief and functional. The profiles' INT-5A overlay confirms that model/example support should demonstrate, not replace, the learner's writing work.

---

### 3.4 Task requirements are stated once at task level, not duplicated in options

**Result: PASS with reservation**

Freer Practice requirements are generally stated at task level. Mild duplication within individual Guided Practice sub-sections (requirements listed in setup then partially repeated in scaffolded prompts) is present but not a significant problem at this level.

---

### 3.5 Choice-list options are parallel in length, detail level, and difficulty

**Result: PARTIAL FAIL**

This remains the most consistent prompt-design problem in the live book and is not addressed in the current profiles. Options within Freer Practice lists mix situation-anchored prompts of varying length with topic-label options, creating unequal apparent difficulty.

The profiles record output expectations (sentence ranges, document types) but do not audit option parallelism within choice lists.

**Required action:** Add a whole-book content flag (or a check item in the relevant units' profiles) requiring that all Freer Practice choice lists be reviewed for option parallelism before the next revision pass. Priority units: 5, 8, 17.

---

### 3.6 Word/sentence count targets are present and appropriate

**Result: PASS**

The profiles improve on the live book here by recording explicit output expectations for every output type in every unit (guided, freer, extended, homework). Sentence-count targets are appropriate for intermediate level. Where the live book leaves some tasks without explicit counts, the profiles supply the intended range — meaning the fix is a content-transfer task rather than a redesign task.

---

### 3.7 Academic framing is avoided

**Result: PASS**

Consistent throughout. Professional framing is used correctly in both the live book and the profiles.

## SECTION 4 — Activity Variety and Progression

---

### 4.1 Activity variety exists across the book

**Result: PARTIAL PASS**

The profiles confirm the consistent unit structure and explicitly note it as an intentional intermediate-level design choice (reducing cognitive overhead). Several profiles include content flags that acknowledge the limited variation but do not mandate introducing new activity types.

Activity types genuinely absent from the intermediate book: sequencing/ordering tasks, error identification tasks before Module 5, data interpretation input, paragraph-level transformation beyond Module 1. The profiles do not plan for these to be introduced.

This is a design decision rather than a gap — the intermediate book is not trying to replicate the advanced book's activity range. The concern is whether the book's narrower range is sufficient to develop the skills it claims to target. For the current level and audience, the consistent structure is defensible, but this should be revisited when planning the next revision pass.

---

### 4.2 Scenario variety exists across the book

**Result: PASS with reservation**

The profiles note the JPO institutional context as the intended scenario world. Several unit profiles carry content flags warning against scenario drift (e.g., Unit 9: "should remain distinct from the delay/change email unit in Module 2"). However, the live book still concentrates heavily on the missing/incorrect document scenario family across Modules 3–4.

The profiles do not include a book-wide scenario-distribution check equivalent to the advanced book's concern about "JPO option clustering." This is a gap.

**Recommended addition:** Add a whole-book scenario-distribution note to the profiles document, similar to the advanced book's N11 check for scenario-family clustering, to prevent over-reliance on the same core scenario type across adjacent units.

---

### 4.3 Progression from controlled to independent production is visible

**Result: PASS**

The profiles confirm and record the scaffolding gradient for every unit (guided → freer → extended → homework). Output length increases across modules as intended. The profiles' output-length expectations provide a clear reference for later QA.

---

### 4.4 Module 6 functions as a genuine capstone

**Result: PASS**

The profiles confirm the capstone logic across Units 19–23. The Unit 23 profile notes the existence of a companion portfolio marking note (`aw-int_unit23_portfolio_marking_note.md`), confirming that P7-equivalent rubric work has been completed for the intermediate book.

---

### 4.5 The book teaches editing and revision as deliberate skills

**Result: PASS**

Module 5 separates accuracy, clarity, and tone editing into three distinct units. The profiles confirm this three-unit structure and record it as an intentional design choice. The INT-5A overlay confirms that editing/revision support is a required element for every unit, not only Module 5 units.

---

### 4.6 Reflection activities are present and focused

**Result: PASS with reservation**

Reflection sections are present in every unit. The profiles confirm them as required structural elements. The reservation from the initial evaluation stands: the reflection questions are formulaic across all 23 units and do not increase in metacognitive complexity in Modules 4–6. The profiles do not flag this for revision, which means it is unlikely to be addressed without an explicit decision to do so.

## PRIORITY FINDINGS — ITEMS REQUIRING ACTION

### Priority 1 — Active gap between profiles and live book

**1.2 / 1.3 — Can-Do statements and activity-purpose maps not yet transferred to live book content**

The planning architecture is complete. The live `### Focus` sections still use instructional bullet lists. This is the most important remaining structural task: transfer the Can-Do statements from the profiles into the live module files. This is a content-transfer task — the content already exists and does not need to be written from scratch.

### Priority 2 — Prompt design issues (not captured in current profiles)

**3.1 / 3.5 — Topic-anchored Freer Practice options and non-parallel choice lists**

These affect approximately one-third of Freer Practice choice lists, concentrated in Units 5–7 (Module 2) and Units 16–18 (Module 5). They are not flagged in the current profiles. Two actions are needed: (a) add content flags to the affected unit profiles, and (b) revise the live content in those units to replace topic labels with situation-anchored prompts and ensure option parallelism.

**Priority units for prompt revision:** 5, 6, 7, 16, 17, 18.

### Priority 3 — Missing profile flags (gaps in the profiles' own coverage)

**2.5 — Units 12 and 13 phrase banks need worked mini-examples**

Not currently flagged. Add to Unit 12 and 13 profiles.

**4.2 — No book-wide scenario-distribution note**

The profiles do not include a check equivalent to the advanced book's N11 scenario-clustering concern. Add a whole-book note to the profiles document.

**4.6 — Reflection questions do not increase in complexity across modules**

Not flagged. Consider adding a note to Module 4–6 profiles that reflection questions should be reviewed for metacognitive depth before the next revision pass.

## OVERALL VERDICT

The intermediate book is in materially better shape than the initial evaluation (conducted without the profiles document) indicated. The two items originally flagged as hard failures — missing Can-Do architecture — are resolved at the planning level. The profiles document is complete, well-structured, and fit for use as the planning source of truth.

The remaining live-content issues fall into two categories. The first is a transfer gap: the profiles contain content (Can-Do statements, output expectations) that has not yet been written into the live book. The second is a prompt design gap: topic-anchored Freer Practice options and non-parallel choice lists in approximately six units are not yet flagged in the profiles and will not be caught by the current content flags without an explicit addition.

The book's language instruction layer remains genuinely strong throughout: the *Why this works* model, Mini contrast pattern, Self-Check gates, and Transfer Extensions are consistent, effective, and supported by the profiles' INT-5A overlay requirements.
