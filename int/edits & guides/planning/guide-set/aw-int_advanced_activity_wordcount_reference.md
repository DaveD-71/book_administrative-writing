> **Purpose of this file**
> This file records the average word counts of the coded activity types in the live advanced textbook.
>
> Use it as a reference during `INT-5A` when judging whether the intermediate book's activity-level support remains materially thinner than the advanced book.
>
> This is a calibration reference, not a direct copying template.

---

## 1. Method

Source used:

- `adv/md/final/modules/aw-adv_mod1_n10.md` to `aw-adv_mod6_n10.md`

Counting rule:

- treat each coded `####` activity block such as `(A1)`, `(B2)`, `(C3)`, etc. as one activity
- count the words inside that activity block only
- stop the count at the next `####`, `###`, or `##` heading

Important interpretation rule:

- these averages are **reference benchmarks**
- they do **not** mean the intermediate book should mechanically copy advanced activity lengths
- they are useful for checking whether the intermediate book remains underdeveloped in support depth at the activity level

## 4. How To Use This Reference During INT-5A

Use this file to ask better calibration questions such as:

- are intermediate model-comparison activities still much thinner than advanced `B1` / `B4` support?
- are intermediate editing routines still underdeveloped relative to `F3`, `G1`, and `G2` functions?
- are freer-writing and extension activities still too light to support the later integrated tasks?
- is the intermediate book short only because it is lower level, or because the activity-level support is still too thin?

Use the averages as **signals**, not commands.

Preferred behavior:

- preserve intermediate readability
- preserve shorter wording where appropriate
- do not preserve avoidable underdevelopment
