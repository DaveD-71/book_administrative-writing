> This brief defines the next execution wave inside `INT-5A`.

---

## 1. Why Wave 4 Exists

The current live totals show that the intermediate book is stronger than before, but the remaining shortfall is no longer best understood as only a set of thin saved activities.

The main discovered pattern is:

- weak rows cluster in support-carrying activity families, especially `C` and `E`
- the pattern intensifies in the later, more complex modules
- the real shortfall is not only length, but underbuilt explanation, model support, guided-to-freer bridging, revision depth, and reader-effect guidance
- broader module-level support has not yet fully caught up with the complexity of the later tasks

Wave 4 therefore shifts from a mainly row-led repair model to a broader module-led rebuild model.

## 4. What Wave 4 Must Add

Wave 4 must not behave like another narrow "make the thin rows longer" pass.

It must add broader support architecture such as:

- fuller model/example comparisons
- stronger guided-to-freer bridges
- explicit reader/job differentiation where one scenario creates multiple writing tasks
- stronger revision walkthroughs and decision logic
- more substantial end-of-module rebuilding and application tasks
- stronger portability of editing and planning habits across modules

The goal is to make the module feel more fully built, not merely more wordy.
