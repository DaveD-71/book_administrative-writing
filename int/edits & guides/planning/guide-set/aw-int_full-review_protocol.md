# Full Review Protocol
## *Administrative Writing, Intermediate*

---

> **Purpose of this document**
> This file defines what the intermediate-book full review is, what it is not, what source files govern it, how findings should be recorded, and what counts as completion.
>
> It is the source-of-truth definition for the post-QA full-review stage.

---

## 1. What Full Review Is

Full review is the first whole-book editorial pass conducted after the intermediate book has cleared its dedicated QA gate.

Its purpose is to evaluate the completed live book as a finished learner-facing product rather than as a draft still undergoing structural repair.

---

## 2. What Full Review Is Not

Full review is not:

- another structural-planning pass by default
- a substitute for QA
- a prompt-repair campaign by default
- a language-instruction redesign pass by default

Reopen earlier stages only if new evidence shows that a significant issue remains live in the current source.

## 4. Recording Rule

When full review begins:

- create a dedicated full-review artifact folder outside planning
- record findings by round
- keep this protocol as the stage-definition source of truth
- keep review findings separate from this protocol

---

## 5. Definition Of Done

The intermediate full-review stage is complete when:

- no unresolved full-review finding remains at blocker level
- remaining lower-severity findings are either corrected or explicitly accepted
- the control documents and live source are aligned closely enough to support answer-key work
- the book is stable enough to justify final support materials
