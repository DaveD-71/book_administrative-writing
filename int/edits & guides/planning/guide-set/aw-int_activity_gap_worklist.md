## 4. Main Development Patterns

The weakest activity classes are not random.

Most repeated gap types:

- controlled-practice blocks that still ask too little of the learner or provide too little instructional framing
- extended writing tasks that are still too short to function as meaningful end-stage output work
- editing activities that still behave like short prompts instead of genuine revision support
- model / explanation activities in later modules that still do not carry enough explanatory weight
- guided activities in the capstone module that remain too light for the document-control demands they are supposed to prepare

The core problem is not only length.

The core problem is that these activities are often still too thin in:

- explanation of why the stronger version works
- task framing
- reader-effect guidance
- revision-support depth
- visible output expectations

## 5. How To Use This Worklist

Use this file during the next `INT-5A` passes as follows:

1. work by module priority, not by scattered one-off edits

4. add development where it improves:
   - explanation

   - output support
   - revision support
