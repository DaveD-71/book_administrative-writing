# Activity Framework
## *Administrative Writing, Intermediate*

---

> **Purpose of this document**
> This file defines the standard intermediate unit architecture, recurring section purposes, and approved structural variations.

---

## 1. Core Principle

The intermediate book should keep a simpler and more teachable identity than the advanced book.

That means:

- fewer structural layers inside each unit
- shorter, clearer instructions
- stronger scaffolding from model to guided writing
- freer writing that is still demanding, but bounded and level-appropriate
- integrated late-book tasks that remain readable and supportable for A2-B1+ learners

---

## 2. Standard Intermediate Unit Functions

The standard intermediate unit should include these functions:

1. focus and setup
2. concept/model section
3. controlled language or practice work
4. guided writing
5. freer writing
6. extended writing or transfer task where appropriate
7. reflection
8. homework

Later integrated units may also include:

- scenario overview handling
- dual-audience writing
- multi-document task logic
- portfolio-style revision work

---

## 3. Intermediate Structural Rules

- Do not import the advanced `A-H` coded architecture wholesale.
- Keep section naming and task sequencing readable for lower-level learners.
- Preserve a visible bridge from explanation to guided work to freer output.
- Avoid piling multiple task purposes into one instruction block.
- Later integrated units may be more complex, but their extra complexity must be prepared by earlier units.

---

## 4. Scenario And Output Rules

- Use concrete, ordinary workplace situations.
- Keep professional realism credible without over-specialization.
- Guided tasks may use sentence counts where that clearly helps learners.
- Freer writing, extended writing, and homework should move toward stable word-range expectations rather than ad hoc sentence counts.
