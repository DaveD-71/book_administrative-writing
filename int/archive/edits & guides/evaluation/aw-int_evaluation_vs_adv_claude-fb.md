# Comparative Evaluation: *Administrative Writing, Intermediate*
## Against the Advanced Book Development Methodology

---

> **Purpose of this document**
> This report evaluates the intermediate book (`aw-int-all.md`) against the four priority dimensions agreed for this pass: (1) pedagogical structure, (2) language instruction quality, (3) prompt and task design, and (4) activity variety and progression. Results are presented as a checklist-style pass/fail assessment, with findings and required actions for each item that fails.

---

## SECTION 1 — Pedagogical Structure (Can-Do Architecture, Activity Sequence)

---

### 1.1 Module introductions are present and coherent

**Result: PASS**

All six modules have a Module Guide section. Each guide states module-level learning outcomes (in "By the end of this module, you should be able to" form) and gives usage instructions. The guides are appropriate for intermediate learners and maintain a consistent format across the book.

---

### 1.2 Unit-level Can-Do statements are present

**Result: FAIL**

The intermediate book uses a `### Focus` section with bullet-point learning objectives (e.g., "Review basic sentence structure," "Practice writing short, clear sentences"). These are instructional descriptions, not Can-Do statements. They tell the learner what the unit will cover, not what the learner will be able to do after completing it.

The advanced book's structural profiles established learner-facing Can-Do statements in the form *"I can…"*, calibrated to testability and distinct from each other within the unit. The intermediate book does not follow this model anywhere across the 23 units.

**Required action:** Convert all unit `### Focus` bullet lists into Can-Do statements using the *"I can…"* format. Each unit should have 3–4 distinct, testable statements. This is the direct equivalent of the advanced book's unit-level Can-Do architecture.

---

### 1.3 Activity-level Can-Do statements are present

**Result: FAIL**

The intermediate book contains no activity-level Can-Do statements. In the advanced book, each activity in the structural profiles carries a single statement connecting the activity's specific function to the unit's learning goals. The intermediate book's activities are titled and sequenced but carry no explicit statement of what the learner should be able to do by completing that activity.

**Required action:** For each unit, write one activity-level Can-Do statement per activity (Warm-Up, Model Check, Language, Guided Practice, Freer Practice, Editing, Transfer Extension, Homework). These do not need to appear in the body text immediately, but they should be recorded in a structural profiles document equivalent.

---

### 1.4 The activity sequence visibly develops the unit goal

**Result: PASS with reservation**

The intermediate book consistently follows this sequence within each unit: Warm-Up → Model Check → Language (1 or more) → Guided Practice → Self-Check → Freer Practice → Editing Practice → Transfer Extension → Reflection → Homework. This is a coherent instructional arc (Activation → Observation → Controlled Practice → Freer Production → Editing → Transfer).

The reservation: the sequence is highly consistent, which is appropriate at intermediate level, but it is also somewhat rigid. Every unit follows the same observable pattern. The advanced book varied activity types deliberately across units to build range. The intermediate book's uniformity is appropriate for lower scaffolding needs, but units in Modules 4–6 that target more complex skills could benefit from modest variation.

---

### 1.5 Module-level progression is coherent (skills build across modules)

**Result: PASS**

The module sequence moves logically from sentence control (Module 1) → email writing (Module 2) → internal documents (Module 3) → external/audience-facing communication (Module 4) → editing and revision (Module 5) → integrated multi-document tasks (Module 6). This mirrors the advanced book's progression architecture and is appropriate.

---

### 1.6 Unit-level skill progression within each module

**Result: PASS with reservation**

Within each module, units address distinct aspects of the module skill (e.g., Module 2: email layout → clarification requests → delays and changes → polite requests). Progression is present and logical.

The reservation is that the step-up in difficulty between units within a module is sometimes modest. The advanced book deliberately increased analytical demand across units within a module. In the intermediate book, Units 5 and 7 (both within Module 2) address related aspects of request writing with similar scaffolding, suggesting scope to differentiate more clearly.

---

### 1.7 Section H ordering is consistent

**Result: PASS**

Every unit ends with a `### Homework` section. The Transfer Extension appears before the Homework task consistently. There is no H1/H2/H3 coding system in the intermediate book, which is appropriate — the advanced book's H-section complexity is a later-level feature. No ordering inconsistencies are present.

---

## SECTION 2 — Language Instruction Quality (N10-Style Teaching Gaps)

The N10 standard requires that language instruction explicitly explains: what a feature is, why it works, when to use it, and how to transfer it. It must go beyond phrase banks, examples, and practice tasks.

---

### 2.1 The principle behind each target feature is named and explained

**Result: PASS**

This is one of the intermediate book's strongest features. Every unit includes a `### Model Check: Why the Stronger [X] Works` section that explicitly states the reason for the improvement. Every major Language section includes a `**Why this works**` block. This mirrors the N10 standard's requirement for explicit teaching of the principle, not just demonstration.

Examples of strong practice:
- Unit 1: "Sentence clarity is not primarily about length — it is about logical load."
- Unit 2: "Professional tone does not mean 'very formal words only.' It means the reader can cooperate without feeling attacked."
- Unit 7: "A polite request does not become weak. It becomes easier for the reader to accept and complete."

---

### 2.2 Conceptual distinctions between related forms are explained

**Result: PASS with reservation**

Some units make good conceptual distinctions. Unit 7's politeness scale (Can you / Could you / Would you be able to / Would it be possible to / I would appreciate it if) is well explained with both examples and notes on appropriate use. Unit 6 distinguishes between delays and changes clearly.

The reservation: some distinctions that matter at this level are not drawn. For example:
- The difference between *because* and *due to* in formal/informal register is not addressed, though both appear in practice items.
- In Unit 3, topic/support/closing structure is explained, but the distinction between a *concluding* sentence and a *forward-linking* sentence is blurred (the model closing sentence in the Language section describes what the paragraph does rather than closing it — an inconsistency that could mislead learners).

---

### 2.3 Worked examples support the explanation where the concept is not self-evident

**Result: PASS**

The intermediate book makes extensive and effective use of `Mini contrast` and `Mini model` blocks. These are one of the strongest N10-style features of the book. They appear consistently in Language sections, Practice instructions, and Guided Practice, making the conceptual principle visible in short, concrete form before the learner attempts practice.

---

### 2.4 Transfer is addressed: learners are helped to carry the concept forward

**Result: PASS**

Each unit includes a `### Transfer Extension` that explicitly asks learners to move the skill to a new context (e.g., a real workplace email, a different audience, a different document type). This is a genuine transfer step, not simply additional practice of the same type. The advanced book used `E Transfer cue` interventions for the same purpose; the intermediate book embeds transfer systematically and more visibly.

---

### 2.5 Phrase banks and useful-language sections are explained rather than listed

**Result: PASS with reservation**

Useful phrase lists appear in most Language sections and are generally accompanied by brief functional labels (e.g., "Giving Information," "Asking for Clarification"). The reserved observation is that some phrase banks in Module 4 (particularly Units 12–14) present phrases without a worked example showing them in a complete message. In the advanced book, phrase banks were considered insufficient without at least one worked mini-example embedded nearby. Units 12 and 13 in the intermediate book would benefit from one short worked mini-inquiry showing context + question + closing operating together.

---

### 2.6 The bridge from explanation to production is visible

**Result: PASS**

The Self-Check Before Freer Practice section in every unit is a genuine bridge. It consistently restates the unit's key criteria as a checklist, requiring the learner to verify readiness before moving to freer production. This is a strong intermediate-level implementation of the N10 bridge requirement.

## SECTION 3 — Prompt and Task Design (PAST-R, Calibration, Parallelism)

---

### 3.1 Writing tasks are situation-anchored, not topic-anchored

**Result: PARTIAL PASS**

Many tasks are situation-anchored and include professional context (e.g., "The applicant sent the wrong form. You need to contact the applicant and ask for the correct form. The deadline is next Friday."). However, a recurring pattern throughout the book is that some Freer Practice options are topic-anchored rather than situation-anchored.

Examples of topic-anchored options that fail this criterion:
- Unit 5 Freer Practice: "Deadline confusion," "Missing attachment," "Conflicting instructions" — these name a topic category, not a workplace situation.
- Unit 8 Freer Practice: "A new colleague needs to know how to prepare a file for internal review" — this is situation-anchored and passes.
- Module 5 Freer Practice options follow the same topic-labelling pattern as Unit 5.

By the advanced book's Prompt Writing Policy standard, the situation is the vehicle, not the point. A prompt naming a category ("Deadline confusion") leaves PAST-R incomplete: no role, no specific audience, no stakes.

**Required action:** Replace topic-label options in Freer Practice lists with situation-anchored prompts. Each option should name a concrete professional event, a reader, and an implicit purpose.

---

### 3.2 PAST-R elements are recoverable from the prompt

**Result: PARTIAL PASS**

Guided Practice tasks are generally well-specified (Purpose, Audience, Subject, Type, Role are usually clear). Freer Practice tasks at the category-label end (see 3.1 above) fail this criterion because Subject and Role are missing.

The Homework tasks are generally stronger than Freer Practice in PAST-R compliance, particularly from Module 3 onwards.

---

### 3.3 The prompt does not pre-solve the writing task

**Result: PASS**

The intermediate book is generally well-calibrated on this dimension. Guided Practice tasks provide structure and a frame, but leave the content decisions to the learner. The Freer Practice tasks give only the situation (where situations are provided). Mini models are brief and functional rather than full worked examples that pre-solve the task.

---

### 3.4 Task requirements are stated once at task level, not duplicated inside individual options

**Result: PASS with reservation**

Freer Practice requirements are usually listed at the task level before the situation options (e.g., "In each email, include: a clear opening / one polite request / one short reason or context sentence / a softened closing line"). This is correct practice per the advanced book's policy.

The reservation: some Guided Practice sections list requirements both in the setup ("Include: Subject line, Greeting, Opening sentence, Body, Closing, Sign-off") and then repeat subsets of these in subsequent parts of the same activity. This creates mild duplication within a single exercise rather than across options.

---

### 3.5 Choice-list options are parallel in length, detail level, and difficulty

**Result: PARTIAL FAIL**

Within Freer Practice lists, parallelism between options is inconsistent. The most common problem is that options mix situation-anchored prompts (longer, more specific) with topic-label options (one or two words). This creates apparent difficulty differences that may lead learners to systematically choose the shorter option as easier, when both should be equally demanding.

Examples:
- Unit 8 Freer Practice mixes well-developed situation options (Option A: 18 words) with less developed ones (Options C and D: shorter, less specified).
- Unit 17 mixes paragraph-level scenarios with short option labels.

**Required action:** Review all Freer Practice choice lists for parallel length and detail level. Each option should be one to two sentences, situation-anchored, and roughly equivalent in scope.

---

### 3.6 Word count targets are present and appropriate

**Result: PASS with reservation**

Most Freer Practice and Homework tasks specify sentence counts or word counts, or specify a range (e.g., "3–4 sentences," "5–7 sentences," "6–8 sentences"). The use of sentence counts rather than word counts is appropriate for intermediate level (unlike the advanced book, where sentence counts were explicitly deprecated in favour of word counts). Sentence counts give simpler, more legible targets for B1 learners.

The reservation: some tasks specify only a range of sentences in the "Include" bullet list without a final word/sentence count summary, making the scope less clear. And Extended Writing Tasks in some units specify only sentence counts without a clear word count floor or ceiling.

---

### 3.7 Academic framing is avoided

**Result: PASS**

The intermediate book consistently uses professional framing ("You need to contact the applicant," "Write to a colleague," "Your supervisor has asked you") rather than academic framing ("Demonstrate your knowledge of," "Using the skills from this unit"). This is correct throughout.

## SECTION 4 — Activity Variety and Progression

---

### 4.1 Activity variety exists across the book

**Result: PARTIAL PASS**

The intermediate book uses a narrower range of activity types than the advanced book. Across all 23 units, the same core sequence repeats: Warm-Up (compare-the-pair) → Model Check → Language (phrase list + rewrite practice) → Guided Practice (scaffold + write) → Self-Check → Freer Practice → Editing Practice → Transfer Extension → Reflection → Homework.

This is pedagogically appropriate for intermediate level — consistency reduces cognitive overhead. However, the book lacks several activity types that could add genuine learning value even at this level:
- No sequencing/ordering tasks (C6 equivalent)
- No error identification and correction tasks (C4 equivalent) until Module 5
- No paragraph-level transformation tasks beyond Module 1
- No data interpretation input (even simple tables could appear by Module 3)

The advanced book deliberately varied B-section engagement (Noticing, Structural Deconstruction, Annotated Model, etc.). The intermediate book uses exclusively the compare-the-pair model for every B-section equivalent.

**Required action:** Not urgent at this stage, but at a future revision pass, introduce modest activity variation in later modules (3–6). Sequencing tasks are particularly underused given Module 3's procedural focus.

---

### 4.2 Scenario variety exists across the book

**Result: PASS with reservation**

Scenarios draw from a limited but consistent professional world: applicant files, document review, system updates, meeting summaries, form submissions, internal notices. This is appropriate and coherent for the JPO institutional context.

The reservation: Modules 3 and 4 reuse the same scenario types with minimal variation (the applicant file problem, the system error, the incorrect form). By Module 4, five consecutive Freer Practice situations involve some version of a missing or incorrect document. Some diversification of professional events (deadline scheduling, personnel changes, guideline updates, coordination meetings) would reduce repetition.

---

### 4.3 Progression from controlled to independent production is visible

**Result: PASS**

The move from Language Practice → Guided Practice → Freer Practice → Editing Practice → Transfer Extension is a genuine scaffolding gradient within each unit. Across modules, the length of Freer Practice outputs increases (3–4 sentences in Module 1 → 5–7 sentences in Modules 4–6), which is appropriate.

---

### 4.4 Module 6 functions as a genuine capstone

**Result: PASS**

Module 6 (Units 19–23) successfully integrates earlier skills into multi-document tasks requiring audience control, consistency, and portfolio revision. The design logic — one scenario, multiple documents, multiple audiences — is a genuine capstone design, not simply harder versions of earlier tasks. Unit 23's portfolio rubric is appropriate in scope and well-aligned to the book's skill strands.

---

### 4.5 The book teaches editing and revision as skills, not only as correction tasks

**Result: PASS**

Module 5 (Units 16–18) explicitly separates accuracy editing, clarity editing, and tone editing into distinct units with different foci. Each unit teaches editing as a deliberate pass, not as a single undifferentiated correction. The Module 5 editing-decision map (accuracy → clarity → tone order) is a strong N10-style metalinguistic teaching tool.

---

### 4.6 Reflection activities are present and focused

**Result: PASS with reservation**

Every unit includes a Reflection section with 3 questions and a Final self-edit prompt. Reflection is treated as a learning activity, not a formality. However, the reflection questions are almost identical in format across all 23 units (what was difficult / which phrase to remember / write one example). By Module 4–6, the questions should increase in metacognitive complexity, asking learners to compare, evaluate, or transfer rather than only recall and note.

## PRIORITY FINDINGS — ITEMS REQUIRING ACTION

### Priority 1 — Hard failures (structural gaps vs. the methodology)

**1.2 — No unit-level Can-Do statements**
All 23 units use instructional bullet lists instead of learner-facing Can-Do statements. This is the most significant structural gap vs. the advanced book's methodology, which treats the Can-Do architecture as the planning source of truth. Recommended fix: rewrite all unit `### Focus` sections into *"I can…"* format (3–4 statements per unit).

**1.3 — No activity-level Can-Do statements**
Activity-level goal tracking does not exist in the intermediate book. Recommended fix: produce a structural profiles document for the intermediate book (equivalent to `aw-adv_unit_structural_profiles_revised.md`) recording one Can-Do statement per activity per unit.

### Priority 2 — Partial failures (quality issues that weaken the book)

**3.1 / 3.2 — Topic-anchored Freer Practice options**
Approximately one-third of Freer Practice choice lists contain topic-label options rather than situation-anchored prompts. This fails the PAST-R standard and reduces the authenticity and demand of the production task.

**3.5 — Non-parallel choice lists**
Options within Freer Practice lists vary significantly in length and specificity. This undermines the principle that all options should be equally demanding and equally usable.

### Priority 3 — Reservations (quality can be improved at next revision)

- 2.2: Some conceptual distinctions missing (register difference between *because* / *due to*; closing vs. forward-linking sentence)
- 2.5: Some Module 4 phrase banks lack a worked mini-example showing integration
- 3.6: Extended writing tasks in some units lack a final word count or sentence count summary
- 4.1: No sequencing tasks, limited error identification tasks, no data interpretation
- 4.2: Scenario variety narrows in Modules 3–4 (over-reliance on the missing/incorrect document scenario family)
- 4.6: Reflection questions do not increase in metacognitive complexity across modules

---

## OVERALL VERDICT

The intermediate book has been successfully developed to a level comparable to the advanced book's N9–N10 standard in most areas. The strongest achievement is the language instruction layer: the *Why this works* model, the Mini contrast pattern, and the Transfer Extension are consistently applied and genuinely effective. Module 6's capstone design is coherent and appropriate.

The two hard failures (missing Can-Do architecture at unit and activity level) are structural gaps that the advanced book treated as foundational — they are the planning source of truth that connects teaching goals to activity design. Addressing these would bring the intermediate book into full alignment with the methodology.

The prompt design issues (topic-anchored options, non-parallel lists) are medium-priority fixes that would improve the authenticity and equity of the production tasks, but they do not compromise the book's overall pedagogical integrity.
