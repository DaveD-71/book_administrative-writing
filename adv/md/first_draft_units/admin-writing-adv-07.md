# MODULE 2. Professional Email Writing for Interagency Contexts

## Unit 7 — Layered Requests: Managing Multi-Part Actions

### Unit Overview

This unit develops learners’ ability to write layered, prioritised, and logically sequenced requests for domestic agencies, colleagues, and international partners. By the end of the lesson, learners will be able to:

- distinguish between primary and secondary requests

- structure multi-part requests to maximise clarity

- explain rationales and sequencing for each requested action

- use diplomatic and conditional phrasing to manage dependency steps

- write multi-step request emails that guide the reader through necessary actions

### Warm-Up

#### Activity 1 — One Request or Multiple?

Identify whether each message contains **one request**, **two requests**, or **three or more**.

1. *Please confirm the applicant’s address and resend the updated file.*

2. *Could you clarify the date in Section 3 and verify whether the term “submission period” aligns with the new guidelines?*

3. *We would appreciate it if you could review the attached document.*

#### Activity 2 — Prioritise the Actions

Rank the following actions from **highest** to **lowest** priority:

- Correct translation inconsistencies

- Confirm whether the previous instruction remains valid

- Provide the updated document version

Discuss: How does order affect clarity?

### Model Text

#### Original (Unstructured)

> *"Please check the translation, send the corrected file, and confirm the date. We need everything as soon as possible."*

#### Improved (Layered and Prioritised)

> *"To proceed with the review, we would appreciate your support with the following points. First, could you confirm whether the date in Section 2.1 should align with the revision issued on 3 August? Once confirmed, we would be grateful if you could review the attached translation for consistency. Finally, if an updated version of the file is available, kindly share it at your convenience. These steps will allow us to complete the review accurately."*

#### Activity — Compare the Versions

Identify differences in:

- sequencing

- clarity of action

- priority order

- tone and reader support

### Noticing Task

Highlight in the improved version:

1. primary request

2. secondary requests

3. sequence markers (*first, once confirmed, finally*)

4. dependency relationships (*once confirmed…*)

5. rationale (*to proceed with the review*)

Discuss: Why does sequencing reduce misunderstanding?

### Language Focus

#### 1. Sequencing and Prioritising Requests

Useful phrases:

- *First, could you…*

- *Once confirmed…*

- *After reviewing…*

- *Finally, we would appreciate it if…*

#### Practice A — Add Sequencing

Rewrite with explicit order:

1. *Check the dates. Review the terminology. Send the updated file.*

#### 2. Conditional and Dependency Language

Useful for requests that rely on prior information.

Examples:

- *Once we receive clarification, we will…*

- *After the timeline is confirmed, we can proceed with…*

#### Practice B — Add Dependencies

Rewrite by showing which action depends on another:

1. *Review the translation. Confirm the meaning of the term first.*

#### 3. Diplomatic Softening for Multi-Part Requests

Useful expressions:

- *We would appreciate your support with the following points…*

- *If possible, could you…?*

- *When convenient, would you be able to…?*

#### Practice C — Add Softening

Rewrite the following:

1. *Send the corrected file and fix the terms.*

### Guided Writing

#### Activity 1 — Rewrite the Original Model

Rewrite the unstructured model into a layered, prioritised request message.
Include:

- 1 primary request

- 2–3 secondary or dependent requests

- sequencing

- diplomatic tone

- next steps

#### Activity 2 — Create a Layered Request

Choose one scenario:

1. You need confirmation of terminology *before* reviewing the translation.

2. An international office must confirm the timeline *before* submitting an updated file.

Write a structured message including:

- context

- sequence of actions

- dependency logic

- reason for priorities

#### Activity 3 — Two Versions

Write:

- one version for a **domestic agency**

- one version for a **foreign office**

Discuss how tone and sequencing shift.

### Freer Writing

#### Task — Write a Multi-Part Request Message (8–12 sentences)

Choose one:

1. Clarifying multiple aspects of a procedural document

2. Requesting several updates needed for a report

3. Coordinating steps for preparing a joint announcement

Your message must include:

- 3–4 sequenced actions

- dependency logic

- diplomatic tone

- rationale

- next steps

### Editing Skills

#### Editing Checklist

Look for:

- confusing order of actions

- missing dependency markers

- unclear or implied priorities

- overly direct requests

- unnecessary repetition

#### Editing Exercise

Rewrite for structure:
*"We need you to correct the dates, confirm which version is valid, and fix the terminology. Then we can proceed."*

Add:

- clarification of sequence

- diplomatic tone

- rationale

### Extension Task

3. Analyse a received email and identify implicit priorities.
