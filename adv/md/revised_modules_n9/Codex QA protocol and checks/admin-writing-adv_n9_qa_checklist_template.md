# N9 QA Checklist Template
## *Administrative Writing, Advanced*

---

> **Purpose of this document**
> Use this checklist for every `N9` module pass. Complete it once before editing to map the task, and again after editing before the module is treated as complete. This checklist exists to prevent reliance on memory and to enforce prompt-policy compliance, especially for scenario choice lists and JPO options.

---

## Part B — Activity-Type Fidelity

For each edited E/H task:

- [ ] The task matches the assigned activity type in the master activity menu
- [ ] The prompt requires the target unit skill
- [ ] The wording fits the level of freedom/scaffolding expected for that activity type
- [ ] The prompt reads like student-facing textbook text, not an implementation note

Special checks:

- [ ] `E1` functions as open scenario-based freer writing, not role-based writing in disguise
- [ ] `E3` gives a meaningful professional role and reader relationship
- [ ] `H3` is independent extended writing, with lighter scaffolding than guided/freer tasks where appropriate

---

## Part C — Prompt-Writing Policy Checks

For each edited E/H prompt:

- [ ] The prompt is situation-anchored, not topic-anchored
- [ ] PAST-R is recoverable
- [ ] The scenario does not pre-solve the learner's analytical work
- [ ] Requirements are stated once at task level
- [ ] The wording shows professional verisimilitude
- [ ] The prompt avoids vague placeholders and abstract category labels
- [ ] The task is clear and usable at B2-B2+ level

---

## Part D — Scenario Choice List Checks

For each task with multiple options:

- [ ] Options are parallel in length
- [ ] Options are parallel in style
- [ ] Options are parallel in detail level
- [ ] Options are parallel in scope/difficulty
- [ ] Options represent genuinely different professional situations
- [ ] No option is just a lightly reworded version of another

---

## Part E — JPO Option Checks

If a JPO option is present:

- [ ] It is labelled exactly `*(JPO context)*`
- [ ] It is one option among equals, not a supplement or special case
- [ ] It matches the other options in length and detail level
- [ ] It requires the same target skill as the other options
- [ ] It uses a genuinely distinct context from options 1–3
- [ ] It does not overlap contextually with options 1–3
- [ ] It does not reduce the total range of learner choice
- [ ] Its role/relationship is not merely a branded duplicate of another option

If any item above fails, the prompt is not ready.

---

## Part F — Section H Checks

- [ ] Section H order is correct: `H2 -> H3 -> H1` when all are present
- [ ] No legacy extension-task block remains in addition to `H1`
- [ ] `H1` is genuinely extensional, not a duplicate of `H3`
- [ ] `H2` has an accessible alternative where needed for self-study or learners without workplace material

---

## Part G — Final Editorial QA

- [ ] Activity-type fidelity confirmed
- [ ] Unit-goal fidelity confirmed
- [ ] Content flags addressed or intentionally deferred with reason
- [ ] Student-facing quality is acceptable
- [ ] No obvious overlap, duplication, or contradiction remains
- [ ] Script validation run after editorial QA
