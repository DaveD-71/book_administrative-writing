# Div Class Reclassification Summary and Decisions

## 1. Purpose of the Reclassification

The original semantic div system evolved during development and gradually became increasingly specific, repetitive, and overlapping. As more instructional patterns were added, the visible labels, Focus lines, and task instructions often began communicating similar information in slightly different ways. This increased visual density, reduced scanability, and weakened the overall editorial rhythm of the pages.

The reclassification project therefore shifted toward a smaller and cleaner semantic system designed to improve pedagogical navigation while reducing redundancy. The objective was not to remove authentic writing-process terminology, but to compress and reorganise the semantic structure more effectively.

| Problem Identified | Intended Improvement |
|---|---|
| Excessive visible label repetition | Smaller semantic system |
| Overlap between labels and instructions | Clearer hierarchy |
| Repetition between labels and Focus lines | Reduced duplication |
| Dense instructional phrasing | Cleaner page rhythm |
| Labels competing with headings | Stronger editorial structure |
| Weak scanability | Better pedagogical navigation |

---

# 2. Core Realization

The page currently often communicates the same information multiple times.

Example structure:

```text
Rewrite Task
Focus: Model Rewriting — Model Revision Practice
Rewrite the original evaluation...
```

In many cases:

- the visible label
- the Focus line
- the instructions

all partially repeat each other semantically.

The revised system aims to separate these functions more clearly.

---

# 3. Revised Hierarchy Model

The revised system separates instructional information into three clearer functional layers. This reduces semantic duplication while preserving pedagogical clarity.

| Layer | Function |
|---|---|
| Visible Label | Broad pedagogical state |
| Focus Line | Specific skill or conceptual focus |
| Instructions | Actual task execution/directions |

This structure allows visible labels to become shorter and cleaner while Focus lines carry nuance and instructions concentrate on task execution.

---

# 4. Major Decision: Preserve Writing-Pedagogy Terminology

An earlier proposal suggested replacing many writing-process terms with broader or more abstract labels. This direction was rejected because the textbook is fundamentally a writing textbook, and many of the existing terms carry genuine pedagogical meaning.

The problem was therefore not the terminology itself, but the accumulation of redundant phrasing and overly stacked semantic framing.

| Retained Terms | Reason for Retention |
|---|---|
| Write | drafting/original production |
| Rewrite | substantial reformulation |
| Revise | improvement/refinement |
| Edit | correction/polishing |
| Structure | organisation/planning |
| Draft / Planning | authentic writing-process stages |

Decision:

Retain authentic writing-process terminology while simplifying the surrounding semantic system.

# 6. Important Semantic Decisions

Several key semantic decisions were made during the reclassification process.

| Term / Area | Decision | Reason |
|---|---|---|
| Notice | Retained strongly | Compact, analysis-oriented, observation-focused, compatible with writing pedagogy |
| Model | Replaced with Example | “Model” implied correctness/ideal targets, while many examples were comparative or intentionally weak |
| Example | Adopted as unified label | More flexible and semantically accurate |
| Rewrite / Revise / Edit | Kept distinct | Represent genuinely different writing processes |

### Rewrite / Revise / Edit Distinction

| Term | Typical Meaning |
|---|---|
| Rewrite | substantial reformulation |
| Revise | improve/refine structure/content |
| Edit | correct/polish language/mechanics |

Maintaining these distinctions strengthened the writing-textbook identity.

---

# 7. Inventory Findings

The inventory review confirmed several important semantic overlaps and helped validate the direction of the simplified system.

| Inventory Finding | Resulting Decision |
|---|---|
| activity-analysis and learn-process often overlapped around guided observation and comparison | Use Notice as a broader umbrella category |
| model, model-good, model-bad, and worked-example overlapped heavily | Replace Model with Example |
| Focus lines had become overloaded with multiple layers of explanation | Simplify Focus lines into shorter noun-phrase-oriented descriptions |

### Focus-Line Simplification Example

| Before | After |
|---|---|
| Focus: Reading-To-Write Input — Source-Based Preparation | Focus: Source synthesis |

The revised direction aims to reduce explanatory density while preserving pedagogical clarity.

# 9. Brief Design-System Summary

Later discussion explored how the simplified div-class system could eventually connect to:

- compact SVG-style category tags
- muted editorial colors
- icon + label systems
- improved visual navigation

Several refinements were discussed around:

- typography balance
- icon size
- tag proportions
- editorial rather than UI-style appearance
- integration alongside textbook body text

However, these discussions were secondary to the main reclassification work.

The primary focus of this document is the restructuring and simplification of the semantic div system within the markdown content itself.
