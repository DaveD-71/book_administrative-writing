
# Final Step 1 Decisions (Approved)

## 1. Activity Headings
- Keep activity subheadings.
- Standardize punctuation and capitalization.
- Use em dash (—) consistently for subtitles.
- Use Title Case consistently.
- Preserve activity-family wording for pedagogical variety.

Canonical pattern:
`#### Activity Type — Subtitle (CODE)`

Examples:
- `#### Practice A — Match The Function (C2)`
- `#### Editing Exercise — Improve Clarity And Flow (G1)`

## 2. Metadata Codes
- Keep codes like `(B1)` in Markdown for QA purposes.
- Remove or visually suppress them in the final Word/PDF output.

## 3. Model Labels
Standardize:
- `Original Text`
- `Revised Text`

Optional subtitle:
- `Original Text — Too Direct`
- `Revised Text — Diplomatic`

Replace all uses of `Version` with `Text`.

## 4. Learning Callouts
All pedagogical notes will use the root label:
`Learn`

Subtypes:
- `learn-process`
- `learn-language`
- `learn-principle`
- `learn-transfer`

Examples:
- `::: learn-process`
- `::: learn-language`

## 5. Placeholder Standardization
Use:
- `Draft`
- `Revised Draft`
- `Planning Notes`
- `Final Version`

## 6. Remove Accidental Emphasis
Reduce redundant inline bold where:
- the surrounding heading/style already provides emphasis
- labels compete visually with headings

## 7. Replace Blockquotes
Replace all instructional blockquotes with semantic Div blocks today.

Do not leave instructional notes as Markdown blockquotes.

## 8. Capitalization
Use Title Case for:
- activity headings
- instructional labels
- major callout labels

## 9. Long-Term Direction
Future direction:
- icons may replace activity-family text labels
- semantic Div blocks become primary pedagogical architecture
