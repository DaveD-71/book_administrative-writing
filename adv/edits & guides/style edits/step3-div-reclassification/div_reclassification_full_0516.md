### Key reclassification patterns

- **notice → learn** (12 cases): Div wrappers that contain explanatory content or worked examples rather than an observation task on given text. Many are "umbrella" divs around nested `language` or `example` content.
- **notice → write** (7 cases): Prediction tasks, data-interpretation tasks, and reflection tasks where the learner produces original text.
- **notice → structure** (2 cases): Sequencing/ordering jumbled items.
- **edit → rewrite** (18 cases): Tasks labelled `edit` where the primary instruction is to rewrite/transform given text rather than to find errors or apply a checklist. Includes Editing Exercise tasks in sections G, and some F-section revision chains.
- **edit → revise** (4 cases): Revision chains where the learner improves their OWN previously drafted text.
- **edit → notice** (2 cases): Track-change simulation tasks where the learner evaluates/accepts/rejects proposed edits to given text.
- **language → learn** (6 cases): Divs that contain teaching explanation rather than a pure reference list of language forms.
- **language → rewrite** (4 cases): Divs where the learner performs a transformation or expansion task on given sentences.
- **language → structure** (3 cases): Phrase bank sorting/organisation tasks.
- **rewrite → structure** (4 cases): Tasks where the learner orders/reorganises given content rather than rewriting prose.
- **rewrite → learn** (2 cases): Div wrappers that contain only teaching explanation, with no rewrite task in the wrapper itself.
- **write → structure** (3 cases): Template-guided tasks where the learner fills a given structural framework.
- **write → rewrite** (1 case): A two-audience adaptation task where the source text is given and must be transformed.
- **write → write** and **notice → notice**: Many classifications confirmed unchanged.
