# N10 Framework — Language Instruction Strengthening
## *Administrative Writing, Advanced*

---

> **Purpose of this document**
> This document defines the `N10` pass for the book. Its purpose is to strengthen actual language teaching across the existing module files after completion of `N9`.
>
> `N10` is not a prompt-design pass and not a structural redesign pass. It is a pedagogical strengthening pass. It addresses the book-wide problem identified in the Step 14 / Step 16 discussions: the book often presents language through exposure, comparison, phrase banks, and practice, but does not consistently explain **what the feature is, why it works, when to use it, or how to transfer it to new writing tasks**.

## 2. Core Problem N10 Must Solve

Across the book, learners are often asked to:

- compare weak and improved texts
- notice useful phrases or structures
- classify or rewrite language
- produce guided and freer writing

But the book does not consistently teach:

- what principle underlies the target skill
- why the improved version works better
- what communicative or rhetorical problem the feature solves
- how to choose between similar forms
- when to transfer the feature to a new writing context

As a result, the book currently supports exposure and practice better than explanation and transfer.

`N10` exists to fix that.

---

## 3. N10 Design Principles

### 3.1 Teach the principle, not just the phrase

Language support should explain the mechanism behind the target feature, not only list expressions or provide examples.

### 3.2 Use the existing learning architecture

Every intervention should strengthen the goals already defined in the structural profiles. `N10` should not invent new unit goals or change the core unit structure unless a local correction is unavoidable.

This means `N10` must actively confirm alignment across three levels:

- module purpose
- unit-level Can-Do statements
- activity-level Can-Do statements

If the current teaching in a unit does not clearly support that three-level alignment, the gap must be noted and the intervention should be designed to repair the teaching connection as far as possible without reopening the structural design.

### 3.3 Intervene where teaching actually happens

The primary target is Section `C`.

Secondary targets are:

- `B1` Compare Two Versions
- `B4` Annotated Model, where present
- selected `B2` / `B3` / `B6` / `B7` tasks where explanation is needed to prepare later writing
- selected `D` activities where a bridge from explanation to guided use is missing

### 3.4 Keep explanations concise but real

The goal is not to turn the book into a theory manual. Explanations should be brief, but they must genuinely teach something.

### 3.5 Use proportional intervention

Not every unit needs the same depth of treatment.

Some units need:

- a short conceptual frame only

Some units need:

- conceptual framing
- worked contrast explanation
- one or more annotated examples
- clarification of distinctions between related forms

### 3.6 Prioritise transfer

The test of a strong `N10` intervention is not whether the learner can complete the immediate task, but whether the learner can understand the principle well enough to use it again in a new administrative writing situation.

### 3.7 Preserve the book’s structure

`N10` should strengthen the existing book, not reopen the Step 14 / Step 15 structural design work.

## 4. Main Targets of the N10 Pass

### 4.1 Section C — Language Focus

This is the main intervention site.

Each significant Language Focus sub-section should normally include:

- a short conceptual framing statement or paragraph
- a clear explanation of the communicative purpose of the feature
- guidance on use, contrast, or choice where ambiguity exists
- at least one worked example where the concept is not self-evident

### 4.2 Section B — Model Text Work

`N10` should strengthen B sections where learners are asked to compare or analyse texts but are not sufficiently shown why the improved version is better.

Main priorities:

- `B1` Compare Two Versions
- `B4` Annotated Model

Typical interventions:

- brief explanation of the key improvement principle
- clearer “why this works” wording
- short commentary on one or two important contrasts

### 4.3 Selected Section D Activities

Only where needed.

Use `D` interventions when learners are asked to apply a feature in guided writing but the bridge from explanation to production is too weak.

Typical interventions:

- one reminder line before the task
- one worked partial example
- one transfer cue linking the task back to the target principle

---

## 5. Standard Intervention Types

`N10` should use a limited set of repeatable intervention types.

### Type A — Conceptual Framing

A short explanatory paragraph placed before a Language Focus practice sequence.

It should answer:

- What is this feature?
- Why does it matter in this writing context?
- When do writers choose it?

### Type B — Contrast Explanation

A short explanation clarifying the difference between two related forms or strategies.

Examples:

- coordination vs. subordination
- directness vs. diplomatic softening
- sequencing vs. mere listing
- issue description vs. blame

### Type C — Worked Example

A concrete example showing the target feature in use, with a brief explanation of why it works.

This is especially important where the unit currently presents a pattern, label, or phrase bank without showing how the feature operates in a realistic sentence or short passage.

### Type D — Model Commentary

A brief note in Section B explaining why an improved model succeeds.

This should move beyond “identify improvements” and help make the principle visible.

### Type E — Transfer Cue

A short bridge statement linking the explanation to a later guided or freer task.

Examples:

- what to watch for
- what choice the learner now needs to make
- what problem the learner should avoid

---

## 6. Intervention Levels

Each unit should be assigned one of three `N10` intervention levels.

### Level 1 — Light Strengthening

Use when the unit already teaches the target skill reasonably well but still needs more explicit explanation.

Minimum expectation:

- conceptual framing added to Section C
- at least one “why this works” clarification in Section B or C

### Level 2 — Standard Strengthening

Use when the unit teaches the skill through practice and comparison, but the principle remains too implicit.

Minimum expectation:

- conceptual framing in all major C sub-sections
- at least one worked example
- at least one strengthened “why this works” element in Section B
- at least one transfer cue before later production

### Level 3 — Deep Strengthening

Use when the unit has conceptual confusion, under-explained contrasts, or a heavy reliance on phrase banks / noticing with too little explanation.

Minimum expectation:

- all Level 2 elements
- explicit clarification of key distinctions
- multiple worked examples or annotations
- targeted repair of concept wording where the current explanation is misleading or incomplete

## 7. Decision Rules for Applying N10

For each unit, ask the following in order:

### 7.0 Do the module, unit, and activity goals align clearly?

Check whether:

- the unit goals clearly support the module’s stated purpose
- the activity-level goals clearly support the unit goals
- the existing teaching in B/C/D visibly develops those goals

If `No`, strengthen the explanation layer so that the intended progression becomes clearer in the unit.
If the mismatch is structural rather than pedagogical, note it for later `N11` review.

### 7.1 Does the unit explicitly teach the principle behind its skill?

If `No`, add conceptual framing.

### 7.2 Does the unit explain why the improved language or model works?

If `No`, strengthen Section B and/or add a worked explanation in Section C.

### 7.3 Does the unit help learners choose between related forms or strategies?

If `No`, add a contrast explanation.

### 7.4 Does the unit give enough concrete demonstration of the target feature?

If `No`, add a worked example.

### 7.5 Is the bridge from explanation to production strong enough?

If `No`, add a transfer cue before a later D/E/H task.

### 7.6 Is the current wording conceptually misleading, inconsistent, or too thin?

If `Yes`, repair the wording directly rather than layering explanation on top of a flawed formulation.

---

## 8. Unit-by-Unit N10 Review Questions

For each unit, review the unit against the following questions:

### 8.1 Alignment

- Does the unit’s teaching clearly support the module’s stated purpose?
- Does the language instruction clearly support the unit-level Can-Do statements?
- Do the activity-level goals clearly build toward the unit-level goals?
- Do the B/C/D activities visibly build toward the unit’s main writing outcome?

### 8.2 Explicit Teaching

- Is the core principle named explicitly?
- Is the target feature explained, not just shown?
- Is the reason for using the feature explained?

### 8.3 Conceptual Clarity

- Are key distinctions explained clearly?
- Are labels accurate and useful?
- Are examples consistent with the explanation?

### 8.4 Worked Support

- Is at least one good example explained where the concept is not obvious?
- Are learners shown what success looks like and why?

### 8.5 Transfer

- Does the learner get help carrying the concept into later writing tasks?
- Is the explanation likely to help in a new scenario, not only in the current exercise?

---

## 9. Minimum Deliverables for Each Unit

A unit can only be treated as `N10-complete` if it includes, at minimum:

1. Clear alignment between the module purpose, the unit goals, and the activity goals as reflected in the teaching sequence
2. A clear conceptual explanation in the main Language Focus area
3. At least one explicit “why this works” element in model-text or language work
4. Sufficient explanation of any key contrast or choice central to the unit
5. At least one worked example where the target concept would otherwise remain implicit
6. A visible bridge from explanation to later writing use

Not every unit must contain the same amount of new text, but every unit must meet all six requirements in some form.

---

## 10. Typical N10 Risk Patterns

The following patterns should trigger stronger intervention:

- phrase banks without explanation
- “identify improvements” tasks without commentary
- structure labels without explanation of function
- contrasts presented without rhetorical distinction
- useful expressions given without when/why guidance
- heavy rewriting practice with no explicit principle
- good/bad examples where the reason for improvement remains implicit
- units where the stated learning goal is broader than what the teaching actually explains

## 11. Suggested Working Process

For each module:

1. Re-read the module introduction and unit Can-Do statements in `adv/edits & guides/planning/aw-adv_unit_structural_profiles_revised.md`
2. Read the current `aw-adv_mod*_n9.md` file
3. Confirm module → unit → activity goal alignment
4. Assign each unit an intervention level (`1`, `2`, or `3`)
5. Identify the missing teaching layer in B/C/D
6. Add the minimum required N10 deliverables
7. Check the revised unit against the review questions in Section 8
8. Record any unit still needing deeper pedagogical repair or structural follow-up for later QA attention

---

## 12. Relationship to N11

`N10` strengthens the book.

`N11` later checks whether that strengthening is sufficient at book level.

So this document is an implementation framework, not the final QA checklist. However, the outcomes of `N10` should be observable enough that `N11` can later verify them.

---

## 13. Immediate Practical Implication

For this book, the likely first-pass priorities are:

- Module 2: diplomatic phrasing, softening, audience choice, and explicit explanation of why phrasing choices work
- Module 3: cohesion principles, connective logic, sequencing, and paragraph-level coherence
- Modules 4–6: explanation of rhetorical logic in more complex document types so learners understand structure, not just format

Module 1 already points to the right model for the kind of conceptual strengthening needed, but even there the framework should be applied systematically rather than assumed complete.
